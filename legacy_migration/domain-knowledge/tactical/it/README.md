# 🛠️ GHO Tactical Domain Knowledge: IT

Este repositorio centraliza el conocimiento técnico profundo del Pod de IT: Implementación, Arquitectura de Código y Estándares de Ingeniería.

## 🗺️ Estructura Táctica
- [**`tactical-domain-map.md`**](./tactical-domain-map.md): Mapa de dominios técnicos, flujos de datos y patrones de arquitectura.
- [**`it-knowledge-passages.psv`**](./it-knowledge-passages.psv): Fragmentos atómicos de implementación técnica (Ej: Git hooks, SSH configs, Go patterns).

## 📐 Lineamientos Tácticos
1. **Implementation Specifics:** A diferencia del conocimiento estratégico, aquí residen los "cómos" técnicos.
2. **Atomic IT Passages:** Fragmentos cortos para que el Backend Dev o el Architect consulten antes de codear.
3. **Reference Analysis:** Todos los pasajes técnicos deben estar alineados con los documentos de análisis en `docs/analysis/`.
